COACH: You hit the nail on the head regarding the bridge table (often called a junction or join table)! That is exactly how relational databases handle a many-to-many relationship. You mentioned using arrays (service[] or an array of strings/entities). While some databases like PostgreSQL do support array data types, the standard relational way to handle this is to use that bridge table with simple, single-value rows. For example, your bridge table might look like this:
CREATE TABLE project_organizations (
    project_id INT REFERENCES service_project(project_id),
    organization_id INT REFERENCES organizations(organization_id),
    PRIMARY KEY (project_id, organization_id)
);
By doing this, if Organization A and Organization B both work on Project 1, you just insert two rows into this new table. It keeps your data incredibly clean and easy to query!
